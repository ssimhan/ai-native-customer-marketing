Key concept

Cognitive
Debt

Build too fast. Auto-approve too long. You lose the theory — and in this world, you're paying for the credits.

"Velocity without understanding isn't progress. It's a bill you pay later."
Technical Debt Lives in the code Messy logic, outdated deps, shortcuts
Cognitive Debt Lives in your brain You stop understanding what it does or how to change it. The code isn't broken — you just can't see it anymore.

— Margaret-Anne Storey, U. of Victoria, Feb 2026

Test-Driven
Development

Write the test your software needs to pass before you write the software.

"If I can't do the minimum viable version of this process manually, I have no business automating it."
01

Define success first. What does done look like?

02

Write the test. What must be true for this to work?

03

Build toward the test — not toward whatever you ask next.
